Here are some common classifications for coding test problems based on the topic or algorithm being tested:

1. Data structures: Problems in this category typically test a candidate's knowledge of common data structures such as arrays, linked lists, stacks, queues, trees, and graphs.

2. Algorithms: This category covers a wide range of problems that test a candidate's knowledge of common algorithms such as sorting, searching, dynamic programming, greedy algorithms, and backtracking.

3. Math and logic: Problems in this category test a candidate's mathematical and logical reasoning skills, such as solving equations, finding prime numbers, or implementing mathematical formulas.

4. String manipulation: These problems test a candidate's ability to manipulate strings and perform operations such as string matching, pattern recognition, and regular expression parsing.

5. Computational geometry: This category covers problems that involve geometric algorithms, such as finding the convex hull of a set of points, calculating the distance between two points in a plane, or determining whether two line segments intersect.

6. Graph theory: These problems test a candidate's knowledge of graph algorithms such as finding shortest paths, minimum spanning trees, and topological sorting.

1. Sorting Algorithms: These algorithms are used to sort a set of data items in a particular order, such as alphabetical or numerical order. Examples include bubble sort, insertion sort, selection sort, merge sort, and quicksort.

2. Searching Algorithms: These algorithms are used to search for a particular item within a dataset. Examples include linear search, binary search, and interpolation search.

3. Dynamic Programming Algorithms: These algorithms are used to solve optimization problems by breaking them down into smaller subproblems and storing the solutions to these subproblems to avoid recomputation. Examples include the Knapsack problem, the Longest Common Subsequence problem, and the Fibonacci sequence.

4. Greedy Algorithms: These algorithms make locally optimal choices at each step in the hope of finding a global optimum. Examples include the Kruskal's algorithm for minimum spanning trees, the Dijkstra's algorithm for shortest paths, and the Huffman coding algorithm for data compression.

5. Backtracking Algorithms: These algorithms are used to find all possible solutions to a problem by incrementally building candidates to the solutions, and abandoning a candidate as soon as it is clear that the candidate cannot possibly be completed to a valid solution. Examples include the N-Queens problem and the Subset Sum problem.

6. Divide and Conquer Algorithms: These algorithms are used to solve problems by breaking them down into smaller subproblems, solving these subproblems recursively, and then combining the solutions to the subproblems to obtain the solution to the original problem. Examples include the Merge Sort algorithm, the Quick Sort algorithm, and the Karatsuba algorithm for multiplication.
