The primary purpose of system calls is to provide a safe and efficient way for user-level processes to access kernel-level resources. For instance, when an application needs to read data from a file, it will call a system call to request that the kernel provides access to the file. The system call will then initiate a context switch, where the kernel takes control of the CPU, performs the requested operation, and returns the result back to the application.

There are several types of system calls, including process control, file management, device management, and network management. Process control system calls are used to create, modify, and terminate processes. File management system calls are used to create, read, write, and delete files. Device management system calls are used to interact with hardware devices such as printers, disks, and keyboards. Network management system calls are used to establish and manage network connections.

Some common examples of system calls include open, read, write, close, fork, exec, wait, and exit. The open system call is used to open a file and returns a file descriptor, which is used to read or write data to the file. The read and write system calls are used to read and write data to a file, respectively. The fork system call is used to create a new process, while the exec system call is used to replace the current process image with a new one. The wait system call is used to wait for a child process to terminate, and the exit system call is used to terminate the current process.

In conclusion, system calls are a crucial aspect of modern operating systems and provide a standardized interface between user-level processes and the kernel.
