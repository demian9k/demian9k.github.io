aws-fundamental-VPC

VPC

Virtual Private Cloud
AWS 리소스간 연결이나 on-premise 서버와 aws의 연동을 정의하는 가상 네트워크.


아래 그림은 여러개의 AWS 계정으로 microservice를 구성한 diagram 이다. 각 시스템에 VPC가 구성되어 연결된 것을 확인할 수 있다.

이미지 출처: aws blog

AWS 리소스는 보안적인 이유로 기본적으로 최초에는 다른 리소스들과 연결되지 않은 상태로 존재하며 하나의 시스템이나 애플리케이션이 되기 위해서는 VPC 라는 네트워크에서 연결되어야 한다.
이 네트워크는 AWS의 물리적인 리소스 위에 논리적으로 구현된 가상 네트워크로 사용자가 자신이 소유한 리소스들을 연결하는 데 사용된다.
그 리소스에는 비즈니스 로직상 의존하는 외부 리소스나 public cloud가 아닌 on-premise 상의 서버들과 연결할 때도 사용된다.


VPC는 아래와 같은 항목을 설정할 수 있다.

항목내용
CIDR 블록CIDR 표현으로 서브넷을 나눌 수 있다.
subnet mask일반적인 subnet masking 표현법을 사용한다.
가용 영역서브넷이 어디에 속하는지를 설정한다.
internet gateway인터넷에 접속하기 위한 인터페이스, VPC가 인터넷에 연결하지 않을 경우 불필요하다.
routing어떤 데이터를 어디에 보낼지 조정한다.

ACL서브넷 단위의 가상 방화벽이다.

VPC 특징과 라우팅 테이블

VPC의 라우팅은 물리적인 라우터가 처리하는 것과 차이가 있다.

  • 소프트웨어 라우터이며, IP 주소를 갖지 않는다.
  • 라우팅 테이블 한 개에 서브넷 여러개를 설정할 수 있다.
  • VPC 1개에 인터넷 게이트웨이는 1로 제한되며, IP 주소를 갖지 않는다.
  • 서브넷 사이 통신은 라우터 없이 직접 통신가능하다.

VPC 기본 사용 절차

  1. VPC 대시보드 접근
  2. VPC 생성 - VPC 이름, CIDR 블록, 테넌시 선택
  3. 서브넷 설정 - 서브넷 이름, target VPC 선택, AZ 선택, CIDR 블록 설정
  4. 인터넷 연결시 설정 - 인터넷 게이트웨이(IGW) 생성, IGW와 VPC 연결, 라우팅 테이블 설정

설정시 고려 사항

  • auto scaling시 IP를 사용하는 리소스들을 위한 IP주소를 확보해야 한다.
  • 보안그룹과 ACL을 설정할 때 인스턴스 용도에 맞는 포트를 설정해야 한다.

기본 VPC

기본적으로 생성되어 있는 VPC이며 서브넷과 IGW가 기본적으로 구성되어 있다. 기본 VPC 범위는 172.31.0.0/16 이다. 기본 서브넷이 AZ 별로 한 개씩 생성되어 있다. 각 서브넷은 /20의 주소 범위로 설정 되어 있다.

서브넷과 DHCP

일반적으로 subnet은 네트워크를 분할해 직접 통신할 수 있는 범위를 좁히고 방화벽을 설정해 보안을 강화하는 것을 목적으로 한다.
AWS에서의 subnet은 여기에 더해 어떤 AZ에 네트워크가 위치할지를 결정한다.

AWS의 서브넷은 /16 으로 제한된다.
VPC에는 DHCP 서버가 동작하여 인스턴스가 추가되면 소속된 서브넷이 가진 IP 중 하나가 할당된다.

인터넷 게이트웨이 (IGW)

AWS 리소스들을 인터넷과 연결한다.

EC2 인스턴스는 private ip만 갖고 있으므로 IGW의 NAT 기능을 통해 인터넷과 연결된다. IGW는 NAT Gateway로서 주소 변환도 수행하지만 인터넷에서 서브넷으로 접속을 못하게 하는 역할도 수행한다.

Security Group, Network ACL

inbound, outbound traffic을 제어한다.

제어하려면 양쪽 모두 설정해야 하며 명시적으로 설정하지 않으면 기본설정이 사용된다.
Security Group, Network ACL은 가상 방화벽이라는 역할은 유사하지만 아래와 같은 차이점이 있다.

항목SecurityGroupNetwork ACL
설정 범위인스턴스에 대해 설정한다.서브넷에 설정한다
규칙allow만 설정할수 있다.allow, deny 설정가능
설정stateful (규칙과 상관없이 반환된 트래픽을 자동 허용한다.)stateless ( 반환된 트래픽을 규칙에 따라 명시적으로 허용한다.)
규칙 적용 순서모든 규칙을 확인하여 트래픽의 allow 여부를 정한다.순서대로 규칙을 처리하면서 allow 여부를 정한다.

네트워크 ACL 하나에 서브넷 여러개를 설정할 수 있지만 서브넷 하나에 네트워크 ACL을 여러개 설정할 수 없다.

Security group은 인바운드를 허가하지 않으며 아웃바운드를 허가하고 네트워크 ACL은 양쪽 모두를 허가하는 것이 기본 설정이다.


VPC Endpoint

AWS에는 VPC를 지원하는 서비스와 지원하지 않는 서비가 있으며 이런 서비스들을 연결하려면 VPC 내부에서 외부로 접속하기 위한 연결점을 제공하는 서비스이다.

VPC 내 서브넷 사이에는 직접 통신이 가능하고 VPC 사이에서도 연결하면 통신이 가능하다.
하지만 VPC 외부의 다른 서비스들과 VPC를 연결하려면 IGW를 이용해 인터넷으로 연결해야 한다.
VPC를 사용하지 않는 대표적인 서비스는 S3가 대표적이다.
IGW로 연결하면 VPC가 외부를 통해 통신하게 되므로 더 연결이 불안정할 수 있고 더 안전하지 못하다는 단점이 있다.

VPC 엔드포인트는 이런 VPC를 사용하지 않는 서비스들과 VPC 내 리소스와의 통신 AWS 내부망을 통해 연결할 수 있는 서비스다.

2가지 종류의 엔드포인트가 있다.

인터페이스 엔드포인트(ENI)

ENI 유형에서는 private ip를 가진 인터페이스 존재하며 각 서비스와 연결하는 출입구 역할을 한다.
AWS PrivateLink라는 방식을 사용해 AWS 외 서비스가 PrivateLink를 지원한다면 사용할 수 있다.

게이트웨이 엔드포인트(GW EP)

GW EP는 서비스 리전 단위로 라우팅 테이블을 설정하는 방식이다. 한번 설정하면 리전의 모든 서비스에 사용할 수 있다.


VPC와 다른 네트워크의 연결

VPC peering
VPC 간 연결하기 위해 활성화해야 되는 기능이다.


AWS Direct Connect\ VPC나 AWS 서비스와 다른 네트워크를 전용선으로 연결하는 서비스이다.

전용선과 AWS를 연결하는 것이므로 물리적인 회선 공사가 필요하다.
또한 AWS에 접속을 처리할 라우터를 설치해야 하므로 비용도 많이 발생한다.

하지만 Direct Connect 공유 설비를 사용한 연결 서비스를 사용하는 방법도 있다.

Direct Connect는 연결 유형으로 private, public이 나눠진다.

private 연결
VPC와 연결하며, VPC에 private gateway를 구축하고 이를 경유하여 통신하는 방법이다.
기본은 1:1 연결이지만 AWS Direct Connect Gateway를 구축하면 이를 분기점으로 1:多(AWS) 로 연결할 수 있다.

public 연결
VPC를 지원하지 않는 서비스(S3 등)은 사용할 수 없으므로 이런 경우 public 연결을 사용한다.
public 연결은 AWS Direct Connect Gateway없이 직접 연결하는 방식이다.


AWS VPN
인터넷 VPN으로 다른 네트워크와 연결하는 방법이다.

기업 내 VPN 지원 라우터가 설치되어 있다면 사용할 수 있다.

VPN을 사용하기 위해서는 VPC에 VPG(Virtual Private Gateway)를 구축한다.
연결할 때는 기업 내 VPN 지원 라우터 이 라우터는 일반적인 VPN을 지원하는 라우터이지만 AWS용어로 Customer Gateway라 한다.
VPG를 생성하면 주요 라우터 기종의 Config file을 제공하므로, 이 파일을 해당 Customer Gateway 역할 라우터에 적용해야 한다.

AWS VPN은 간단한 방법이지만 인터넷을 사용하므로 network 품질과 속도를 보장하지 않는다.


Transit Gateway

전송 게이트웨이는 VPC나 on-premise network(AWS DC GW나 VPC Connection)를 하나로 묶어 서로 연결하는 접속접을 제공하는 서비스이다. 서로 다른 AWS 계정도 연결할 수 있다.

네트워크의 연결을 설계할 때 수가 많아지면 VPC Peering, DC GW를 개별적으로 구축해야 하므로 복잡한 문제가 될수 있는데 Transit Gateway는 이런 네트워크 여러개를 중앙 거점으로 집약하여 통신 경로를 통합적으로 처리하는 서비스이다.
