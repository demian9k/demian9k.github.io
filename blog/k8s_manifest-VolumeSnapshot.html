K8S manifest - VolumeSnapshot

VolumeSnapShot은

관련 리소스는 3가지다.

  • VolumeSnapshot
  • VolumeSnapshotClass
  • VolumeSnapshotContent

VolumeSnapshotClass spec을 기준으로 사용자가 VolumeSnapshot 리소스를 apply(적용)하면 VolumeSnapshotContent가 생성된다.

이 관계는 StorageClass를 기준으로, 사용자가 PVC로 요청하고, PV가 실제로 할당되는 관계와 유사하다.

GCE PersistentDisk 를 사용해 VolumeSnapshot을 생성하려면 아래와 절차로 진행한다.

#!/bin/bash

if [ -z "${PROJECT}" ]; then
  echo please set PROJECT env
fi

GCE_PD_SA_NAME=gce-pd-csi-sa

# Install CSI Common Component
git clone https://github.com/kubernetes-csi/external-snapshotter -b v2.1.1
kubectl apply -f external-snapshotter/config/crd
kubectl apply -f external-snapshotter/deploy/kubernetes/ -R

# Install GCE PersistentDisk CSI Driver
git clone https://github.com/kubernetes-sigs/gcp-compute-persistent-disk-csi-driver
git checkout ac1f8c -C gcp-compute-persistent-disk-csi-driver/
kubectl create namespace gce-pd-csi-driver
kubectl apply -k ./gcp-compute-persistent-disk-csi-driver/deploy/kubernetes/overlays/stable



IAM_NAME=${GCE_PD_SA_NAME}@${PROJECT}.iam.gserviceaccount.com

gcloud iam service-accounts create ${GCE_PD_SA_NAME} --project ${PROJECT}

gcloud iam roles create gcp_compute_persistent_disk_csi_driver_custom_role --quiet \
  --project ${PROJECT} --file ./gcp-compute-persistent-disk-csi-driver/deploy/gcp-compute-persistent-disk-csi-driver-custom-role.yaml

gcloud projects add-iam-policy-binding ${PROJECT} --member serviceAccount:${IAM_NAME} --role roles/compute.storageAdmin
gcloud projects add-iam-policy-binding ${PROJECT} --member serviceAccount:${IAM_NAME} --role roles/iam.serviceAccountUser
gcloud projects add-iam-policy-binding ${PROJECT} --member serviceAccount:${IAM_NAME} --role projects/${PROJECT}/roles/gcp_compute_persistent_disk_csi_driver_custom_role

gcloud iam service-accounts keys create ./cloud-sa.json --iam-account ${IAM_NAME} --project ${PROJECT}

kubectl create namespace gce-pd-csi-driver
kubectl -n gce-pd-csi-driver create secret generic cloud-sa --from-file=./cloud-sa.json

rm -f ./cloud-sa.json
rm -rf ./external-snapshotter ./gcp-compute-persistent-disk-csi-driver
  1. install-gcp-csi 스크립트 실행
  • CSI Common Component 및 GCE PersistentDisk CSI Driver 설치
  • GCP의 ServiceAccount 생성하고 그 인증정보를 시크릿 리소스로 등록한다.
  1. PVC에서 참조해야 하는 Stroage class 작성한다.
  2. VolumeSnapshotClass를 작성한다.
  3. 2에서 선언한 storage class를 spec으로 하여 PVC를 작성한다.
  4. VolumeSnapshot을 작성한다. volumeSnapshotClass와 PVC를 매핑한다.
# pvc 에서 사용해야 하는 스토리지 클래스를 먼저 선언한다.
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: sample-csi-storageclass
provisioner: pd.csi.storage.gke.io #
parameters:
  type: pd-standard
reclaimPolicy: Delete
volumeBindingMode: Immediate
allowVolumeExpansion: true

---
apiVersion: snapshot.storage.k8s.io/v1beta1
kind: VolumeSnapshotClass
metadata:
  name: sample-csi-volumesnapshotclass
driver: pd.csi.storage.gke.io
deletionPolicy: Delete

---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: source-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: sample-csi-storageclass
  resources:
    requests:
      storage: 1Gi
---
# 이 Pod는 StorageClass, PersistentVolumeClaim 만으로 생성되었다.
apiVersion: v1
kind: Pod
metadata:
  name: source-pod
spec:
  containers:
    - name: tools-container
      image: amsy810/tools
      command: ['/bin/sh', '-c']
      args: ['date > /data/time.txt && sleep infinity']
      volumeMounts:
        - name: data-volume
          mountPath: /data
  volumes:
    - name: data-volume
      persistentVolumeClaim:
        claimName: source-pvc
        readOnly: false
---
# VolumeSnapshot을 생성하면 해당 시점의 PVC에 연결되는 PV의 스냅샷이 생성된다.
# 생성된 스냅샷의 정보는 VolumeSnapshotContent라는 리소스로 k8s에 등록된다.
apiVersion: snapshot.storage.k8s.io/v1beta1
kind: VolumeSnapshot
metadata:
  name: source-pvc-snapshot
spec:
  volumeSnapshotClassName: sample-csi-volumesnapshotclass
  source:
    persistentVolumeClaimName: source-pvc
---

kubectl get volumesnapshot


kubectl get volumesnapshotcontents

생성된 VolumeSnapshot 을 PVC에서 사용하여 PV 리소스 만들기 dataSource.kind를 VolumeSnapshot 유형으로 지정한다.

여기서 생성하는 볼륨 크기는 스냅샷보다 크게 설정할 수 있다. 스냅샷은 source-pvc(1Gi)에서 생성되었지만, 이 PVC는 3Gi를 할당하고 있다. 이 PVC를 apply하면 PV가 생성된다.

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: restored-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: sample-csi-storageclass
  resources:
    requests:
      storage: 3Gi
  dataSource:
    kind: VolumeSnapshot
    name: source-pvc-snapshot
    apiGroup: snapshot.storage.k8s.io

스냅샷으로 생성된 pvc를 Pod에서 일반적인 pvc처럼 사용한다.

apiVersion: v1
kind: Pod
metadata:
  name: restored-pod
spec:
  containers:
    - name: tools-container
      image: amsy810/tools
      volumeMounts:
        - name: data-volume
          mountPath: /data
  volumes:
    - name: data-volume
      persistentVolumeClaim:
        claimName: restored-pvc
        readOnly: false
