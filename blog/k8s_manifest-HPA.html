K8S manifest - HPA

Horizontal Pod Autoscaler

처리량이 늘어야 할 때를 대비하여 Deployment의 Pod 갯수를 유동적으로 관리하는 기능 제공

HPA가 작동하려면 pod가 어느 정도 자원이 사용되는지 확인할 수 있어야 하는데, 기본적으로는 이 자원 사용량 체크를 하지 않고 있다.

Metrics-Server라는 서버가 활성화되면 Pod자원의 metric을 collect 하며 아래 기능을 사용할 수 있게 된다.

  • Metrics-API 제공
  • kubectl top pods 명령어를 통해서도 metric 확인
  • HPA, VPA를 작동시키는 데 필요한 자원을 참조

Metric-Server 공식문서 내용

공식문서에서 Metric-Server가 제공하는 것은 다음과 같다.

  • 대부분의 클러스터에서 작동하는 단일 배포(요구 사항 참조)
  • 15초마다 메트릭을 수집하는 빠른 자동 확장.
  • 클러스터의 각 노드에 1밀리 코어의 CPU와 2MB의 메모리를 사용하는 리소스 효율성.
  • 최대 5,000개의 노드 클러스터까지 확장 가능.

메트릭 서버를 사용할 수 있는 대상은 다음과 같습니다:

  • CPU/메모리 기반 HPA
  • 컨테이너에 필요한 리소스 자동 adjusting/suggesting(VPA)

다음의 경우는 메트릭 서버를 사용하기에 적절하지 않다

  • 쿠버네티스가 아닌 클러스터
  • 리소스 사용량 메트릭이 정확해야 하는 경우
  • CPU/메모리가 아닌 다른 리소스를 기반으로 하는 HPA

https://github.com/kubernetes-sigs/metrics-server

테스트 설치

kubectl create -f metrics-server.yaml


#공식
kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

테스트를 위해 metrics-server.yaml 에서 수정된 부분

## sig의 원본 yaml 에서 수정된 부분 시작
containers:
  - name: metrics-server
    image: k8s.gcr.io/metrics-server-amd64:v0.3.6
    args:
      # Manually Add for lab env(Sysnet4admin/k8s)
      # skip tls internal usage purpose
      # TLS 인증 무시
      - --kubelet-insecure-tls
      # kubelet could use internalIP communication
      # kubelet이 내부 주소를 우선 사용하게 한다.
      - --kubelet-preferred-address-types=InternalIP
      - --cert-dir=/tmp
      - --secure-port=4443
## sig의 원본 yaml 에서 수정된 부분 끝

#해당 이미지로 deployment 생성
kubectl create deployment hpa-hname-pods --image=sysnet4admin/echo-hname # nginx

#서비스로 80포트 expose
kubectl expose deployment hpa-hname-pods --type=LoadBalancer --name=hpa-hname-svc --port=80

echo-hname deployment의 resource를 수정하여 배포 책에서는 kubectl edit deployment hpa-hname-pods로 수정 m은 milliunits의 약어로 1000m은 1개의 cpu가 되고 10m은 파드의 CPU 0.01 사용을 기준으로 파드를 증설하도록 설정 50m은 한쪽 파드로 부하가 몰릴 경우를 대비한 CPU 사용 제한을 부여한 것.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: echo-hname
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: echo-hname
          image: sysnet4admin/echo-hname
          resources:
            requests:
              cpu: '10m'
            limits:
              cpu: '50m'

위 Deployment로 apply 하면 새로운 파드가 생성된다.

아래처럼 autoscale로 자동으로 scale이 수행되도록 할 수 있다. min은 최소 pod 갯수, max는 최대 파드의 갯수이다. cpu-percent는 CPU 사용량이 50%를 넘으면 autoscale 하겠다는 의미이다.

#deployment에 대해 autoscale 설정
kubectl autoscale deplyment hpa-hname-pods --min=1 --max=30 --cpu-percent=50

#aggregated-metrics-reader.yaml
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: system:aggregated-metrics-reader
  labels:
    rbac.authorization.k8s.io/aggregate-to-view: 'true'
    rbac.authorization.k8s.io/aggregate-to-edit: 'true'
    rbac.authorization.k8s.io/aggregate-to-admin: 'true'
rules:
  - apiGroups: ['metrics.k8s.io']
    resources: ['pods', 'nodes']
    verbs: ['get', 'list', 'watch']

#auth-delegator.yaml
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: metrics-server:system:auth-delegator
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: system:auth-delegator
subjects:
  - kind: ServiceAccount
    name: metrics-server
    namespace: kube-system

#metrics-apiservice.yaml
---
apiVersion: apiregistration.k8s.io/v1beta1
kind: APIService
metadata:
  name: v1beta1.metrics.k8s.io
spec:
  service:

    namespace: kube-system
  group: metrics.k8s.io

  insecureSkipTLSVerify: true
  groupPriorityMinimum: 100
  versionPriority: 100

#metrics-server-deployment.yaml
---
apiVersion: v1
kind: ServiceAccount
metadata:
  name: metrics-server
  namespace: kube-system
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: metrics-server
  namespace: kube-system
  labels:
    k8s-app: metrics-server
spec:
  selector:
    matchLabels:
      k8s-app: metrics-server
  template:
    metadata:
      name: metrics-server
      labels:
        k8s-app: metrics-server
    spec:
      serviceAccountName: metrics-server
      volumes:
        # mount in tmp so we can safely use from-scratch images and/or read-only containers
        - name: tmp-dir
          emptyDir: {}
      ## sig의 원본 yaml 에서 수정된 부분 시작
      containers:
        - name: metrics-server
          image: k8s.gcr.io/metrics-server-amd64:v0.3.6
          args:
            # Manually Add for lab env(Sysnet4admin/k8s)
            # skip tls internal usage purpose
            # TLS 인증 무시
            - --kubelet-insecure-tls
            # kubelet could use internalIP communication
            # kubelet이 내부 주소를 우선 사용하게 한다.
            - --kubelet-preferred-address-types=InternalIP
            - --cert-dir=/tmp
            - --secure-port=4443
          ## sig의 원본 yaml 에서 수정된 부분 끝
          ports:
            - name: main-port
              containerPort: 4443
              protocol: TCP
          securityContext:
            readOnlyRootFilesystem: true
            runAsNonRoot: true
            runAsUser: 1000
          imagePullPolicy: Always
          volumeMounts:
            - name: tmp-dir
              mountPath: /tmp
      nodeSelector:
        beta.kubernetes.io/os: linux
        kubernetes.io/arch: 'amd64'

#metrics-server-service.yaml
---
apiVersion: v1
kind: Service
metadata:
  name: metrics-server
  namespace: kube-system
  labels:
    kubernetes.io/name: 'Metrics-server'
    kubernetes.io/cluster-service: 'true'
spec:
  selector:
    k8s-app: metrics-server
  ports:
    - port: 443
      protocol: TCP
      targetPort: main-port
